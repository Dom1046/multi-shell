# Multi-Shell - 제품 문서

## 프로젝트 개요

### 미션

AI Claude Code 사용 시 여러 셸 창을 동시에 관리해야 하는 불편함을 해소하여, 개발자가 하나의 통합 인터페이스에서 모든 셸 세션을 효율적으로 제어할 수 있도록 한다.

### 비전

Claude Code와 함께하는 AI 기반 개발 워크플로우에서 멀티 셸 관리의 표준 도구가 되는 것. 개발자가 컨텍스트 전환 없이 여러 작업을 병렬로 수행할 수 있는 환경을 제공한다.

---

## 핵심 사용자 및 사용 시나리오

### 주요 페르소나

**페르소나 1: AI 개발자 (Primary)**

- Claude Code를 일상적으로 사용하는 개발자
- 여러 에이전트/작업을 병렬로 실행하면서 각각의 출력을 모니터링해야 함
- 현재 tmux/screen을 사용하지만 AI 워크플로우에 최적화되지 않아 불편함을 느낌

**페르소나 2: 풀스택 개발자**

- 프론트엔드, 백엔드, 데이터베이스 등 여러 서버를 동시에 실행하는 개발자
- 각 서비스의 로그를 한눈에 보면서 개발하고 싶어함
- 셸 간 데이터 복사/이동이 빈번함

### 핵심 시나리오

1. **병렬 AI 작업 모니터링**: Claude Code 에이전트 3-4개를 동시에 실행하고, 각각의 진행 상황을 실시간으로 확인
2. **개발 서버 통합 관리**: 프론트엔드 dev 서버, 백엔드 API 서버, DB 마이그레이션 등을 하나의 화면에서 관리
3. **빠른 셸 전환**: 키보드 단축키로 셸 간 즉시 전환 및 출력 확인
4. **세션 저장/복원**: 작업 환경을 저장하고 다음 날 동일한 구성으로 복원

## 핵심 기능

### MVP (최소 기능 제품)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 멀티 패널 뷰 | 여러 셸을 분할 화면으로 동시 표시 | 높음 |
| 셸 생성/종료 | 새 셸 인스턴스 생성 및 종료 관리 | 높음 |
| 키보드 네비게이션 | 단축키로 패널 간 이동 및 포커스 전환 | 높음 |
| 명령어 실행 | 특정 패널에 명령어 전송 및 실행 | 높음 |
| 출력 스크롤백 | 각 셸의 출력 히스토리 스크롤 및 검색 | 중간 |

### 확장 기능 (Post-MVP)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 레이아웃 프리셋 | 자주 쓰는 패널 배치를 저장/불러오기 | 중간 |
| 세션 영속성 | 프로그램 재시작 시 세션 복원 | 중간 |
| 셸 간 데이터 파이프 | 한 셸의 출력을 다른 셸의 입력으로 전달 | 낮음 |
| 알림/하이라이트 | 특정 키워드 감지 시 시각적 알림 | 낮음 |
| 프로필 관리 | 프로젝트별 셸 구성 프로필 | 낮음 |

## 비즈니스 목표 및 성공 지표

### 성공 지표 (KPI)

| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| 셸 전환 시간 | 0.5초 이내 | 단축키 입력부터 포커스 전환까지 |
| 동시 셸 지원 수 | 최소 8개 | 성능 저하 없는 동시 실행 수 |
| 메모리 사용량 | 셸 당 50MB 이하 | 프로세스 메모리 모니터링 |
| 시작 시간 | 1초 이내 | 프로그램 실행부터 첫 셸 준비까지 |

### 차별화 요소

- **AI 워크플로우 최적화**: Claude Code/AI 에이전트 환경에 특화된 멀티 셸 관리
- **경량 설계**: tmux보다 간단한 설정, 즉시 사용 가능한 기본값 제공
- **크로스 플랫폼**: Windows, macOS, Linux 모두 지원
- **현대적 UX**: TUI(Terminal UI) 기반의 직관적 인터페이스

### 경쟁 제품 대비 포지셔닝

| 대안 | 장점 | 단점 (multi-shell이 해결) |
|------|------|---------------------------|
| tmux | 강력하고 안정적 | 설정 복잡, 학습 곡선 높음, Windows 미지원 |
| screen | 가볍고 보편적 | 기능 제한, 분할 화면 불편 |
| Windows Terminal 탭 | Windows 네이티브 | 동시 확인 불가, 탭 전환 필요 |
| iTerm2 분할 | macOS 최적화 | macOS 전용, AI 워크플로우 미고려 |
