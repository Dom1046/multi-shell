# SPEC-MULTI-003: 수락 기준

## 관련 SPEC

- SPEC ID: SPEC-MULTI-003
- 제목: 출력 스크롤백 및 설정 관리

---

## 수락 기준

### AC-201: 스크롤백 버퍼 동작

**Given** 셸에서 대량의 출력이 생성된 상태에서
**When** 사용자가 Normal 모드에서 PageUp 또는 Ctrl+U를 누르면
**Then** 패널 출력이 위로 스크롤되어 이전 출력을 확인할 수 있어야 한다

검증 방법:
- 100줄 이상 출력 후 스크롤 동작 확인
- PageUp/PageDown으로 반 화면씩 스크롤
- Ctrl+U/D로 반 화면씩 스크롤
- 스크롤 상단 도달 시 더 이상 스크롤하지 않음

---

### AC-202: 자동 스크롤 제어

**Given** 스크롤이 위로 올라간 상태에서
**When** 셸에서 새 출력이 생성되면
**Then** 현재 스크롤 위치가 유지되어야 한다 (자동 스크롤하지 않음)

**Given** 스크롤이 위로 올라간 상태에서
**When** 사용자가 맨 아래로 스크롤하면
**Then** 자동 스크롤이 재개되어 새 출력이 실시간으로 표시되어야 한다

검증 방법:
- 스크롤 위치 유지 확인 (새 출력에도 현재 뷰 변경 없음)
- End 키 또는 맨 아래 스크롤 후 자동 스크롤 재개 확인

---

### AC-203: 스크롤백 버퍼 크기 제한

**Given** 스크롤백 버퍼가 max_lines(기본 10,000)에 도달한 상태에서
**When** 새 출력 줄이 추가되면
**Then** 가장 오래된 줄이 제거되고 새 줄이 추가되어야 한다

검증 방법:
- 10,000줄 이상 출력 후 메모리 사용량 안정 확인
- 가장 오래된 줄 접근 불가 확인
- 버퍼 크기가 max_lines를 초과하지 않음

---

### AC-204: 출력 검색

**Given** Normal 모드에서 출력 히스토리가 있는 상태에서
**When** 사용자가 `/`를 누르고 검색어를 입력한 후 Enter를 누르면
**Then** 일치하는 텍스트가 하이라이트되고 첫 번째 결과로 스크롤해야 한다

검증 방법:
- 검색 입력바 하단 표시 확인
- 검색어 일치 텍스트 하이라이트 확인
- n키로 다음 결과, N키로 이전 결과 이동
- 결과 없을 시 "No results" 메시지 표시
- Escape로 검색 종료 및 하이라이트 해제

---

### AC-205: 설정 파일 로드

**Given** `{config_dir}/multi-shell/config.toml` 파일이 존재하는 상태에서
**When** 애플리케이션이 시작되면
**Then** 설정 파일의 값이 적용되어야 한다

검증 방법:
- scrollback_lines 설정 변경 후 버퍼 크기 확인
- max_shells 설정 변경 후 최대 셸 수 확인
- default_shell 설정 변경 후 기본 셸 확인

---

### AC-206: 설정 파일 없음/오류 시 기본값 적용

**Given** 설정 파일이 없거나 잘못된 형식인 상태에서
**When** 애플리케이션이 시작되면
**Then** 기본값으로 정상 실행되어야 한다

검증 방법:
- 설정 파일 삭제 후 정상 시작 확인
- 잘못된 TOML 형식 파일로 시작 시 경고 로그 + 정상 실행
- 범위 초과 값 (scrollback_lines = -1) 시 기본값 적용

---

### AC-207: CLI 인자 오버라이드

**Given** 설정 파일에 `scrollback_lines = 5000`이 설정된 상태에서
**When** `multi-shell --scrollback 20000`으로 실행하면
**Then** 스크롤백 버퍼 크기가 20,000으로 적용되어야 한다

검증 방법:
- CLI 인자가 설정 파일보다 우선하는지 확인
- --shell, --num, --scrollback 각각 오버라이드 검증
- --config로 대체 설정 파일 로드 검증
- --help, --version 정상 출력 확인

---

### AC-208: 스크롤백 메모리 효율

**Given** 8개 셸이 각각 10,000줄의 스크롤백 버퍼를 사용하는 상태에서
**When** 일반적인 사용 패턴으로 운영하면
**Then** 스크롤백으로 인한 추가 메모리가 셸당 5MB를 초과하지 않아야 한다

검증 방법:
- 8개 셸 x 10,000줄 기준 총 메모리 40MB 이하
- 장시간 사용 시 메모리 증가 없음 (링 버퍼 동작 확인)
