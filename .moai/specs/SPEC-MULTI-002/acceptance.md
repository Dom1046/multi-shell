# SPEC-MULTI-002: 수락 기준

## 관련 SPEC

- SPEC ID: SPEC-MULTI-002
- 제목: 멀티패널 레이아웃 및 키보드 네비게이션

---

## 수락 기준

### AC-101: 새 셸 패널 생성

**Given** Normal 모드에서 셸이 1개 실행 중인 상태에서
**When** 사용자가 `n` 키를 누르면
**Then** 새 셸 패널이 생성되고 화면이 좌우 분할로 변경되어야 한다

검증 방법:
- 패널 분할 전환 시각적 확인
- 새 셸의 프롬프트 표시 확인
- 기존 셸의 출력이 유지되는지 확인

---

### AC-102: 패널 포커스 이동

**Given** Normal 모드에서 2개 이상의 셸이 실행 중인 상태에서
**When** 사용자가 `h/j/k/l` 또는 방향키를 누르면
**Then** 해당 방향의 패널으로 포커스가 이동하고 테두리 색상이 변경되어야 한다

검증 방법:
- 포커스 패널의 테두리가 강조 색상(Cyan)으로 표시
- 비활성 패널의 테두리가 어두운 색상(DarkGray)으로 표시
- 상태바의 셸 번호 업데이트 확인
- 경계 패널에서 더 이동 시 순환(wrap) 또는 유지 동작 확인

---

### AC-103: 셸 종료 및 레이아웃 재계산

**Given** Normal 모드에서 3개의 셸이 실행 중인 상태에서
**When** 사용자가 `x` 키를 누르면
**Then** 활성 셸이 종료되고 남은 2개 패널로 레이아웃이 재계산되어야 한다

검증 방법:
- 셸 프로세스 종료 확인
- 레이아웃이 2패널 배치로 전환
- 포커스가 인접 패널로 자동 이동
- 0.5초 이내에 레이아웃 전환 완료

---

### AC-104: Normal/Insert 모드 전환

**Given** Normal 모드 상태에서
**When** 사용자가 `i` 또는 `Enter`를 누르면
**Then** Insert 모드로 전환되고 이후 키 입력이 활성 셸로 전달되어야 한다

**Given** Insert 모드 상태에서
**When** 사용자가 `Escape`를 누르면
**Then** Normal 모드로 복귀하여 단축키가 다시 활성화되어야 한다

검증 방법:
- 모드 전환 시 상태바 텍스트 변경 확인 ([NORMAL] <-> [INSERT])
- Insert 모드에서 일반 키 입력이 셸로 전달되는지 확인
- Normal 모드에서 `n`, `x` 등 단축키가 셸 관리 명령으로 처리되는지 확인
- Insert 모드에서 `n` 키가 셸로 전달되는지(단축키가 아닌) 확인

---

### AC-105: 숫자키 직접 패널 선택

**Given** Normal 모드에서 4개의 셸이 실행 중인 상태에서
**When** 사용자가 `3`을 누르면
**Then** 3번 패널로 포커스가 즉시 이동해야 한다

검증 방법:
- 1-8 숫자키 각각 테스트
- 존재하지 않는 번호 입력 시 무시되는지 확인
- 포커스 이동 시간 0.5초 이내

---

### AC-106: 상태바 표시

**Given** 애플리케이션이 실행 중인 상태에서
**When** 모드 또는 활성 셸이 변경되면
**Then** 상태바에 현재 모드, 셸 번호, 셸 이름, 관련 단축키가 표시되어야 한다

검증 방법:
- Normal 모드: `[NORMAL] Shell 2/4 | PowerShell | n:New x:Close i:Insert Ctrl+Q:Quit`
- Insert 모드: `[INSERT] Shell 2/4 | PowerShell | Esc:Normal`
- 셸 생성/종료 시 셸 수 업데이트 확인
- 포커스 변경 시 셸 번호 업데이트 확인

---

### AC-107: 최대 8개 셸 동시 실행

**Given** Normal 모드에서 셸을 계속 생성하는 상태에서
**When** 8번째 셸이 생성되면
**Then** 2x4 격자 레이아웃으로 모든 셸이 표시되어야 한다

**When** 9번째 셸 생성을 시도하면
**Then** 생성이 거부되고 상태바에 "최대 셸 수 도달" 메시지가 표시되어야 한다

검증 방법:
- 8개 셸 동시 실행 시 CPU 사용률 50% 미만
- 8개 셸 동시 실행 시 셸당 추가 메모리 20MB 이하
- 모든 패널에서 입력/출력 정상 동작

---

### AC-108: 마지막 셸 종료 시 앱 종료

**Given** 셸이 1개만 실행 중인 상태에서
**When** 사용자가 `x`를 눌러 마지막 셸을 종료하면
**Then** 애플리케이션이 정상적으로 종료되어야 한다

검증 방법:
- 터미널 상태 정상 복원 확인
- 고아 프로세스 없음 확인
