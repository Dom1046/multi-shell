# SPEC-MULTI-001: 수락 기준

## 관련 SPEC

- SPEC ID: SPEC-MULTI-001
- 제목: MVP - 코어 엔진 및 단일 셸 실행

---

## 수락 기준

### AC-001: 애플리케이션 시작 및 셸 생성

**Given** 사용자가 multi-shell 바이너리를 실행하면
**When** 애플리케이션이 초기화되면
**Then** 500ms 이내에 OS 기본 셸이 포함된 TUI 화면이 표시되어야 한다

검증 방법:
- 시작 시간 측정 (바이너리 실행 ~ 첫 프레임 렌더링)
- 셸 프롬프트 표시 확인
- Windows: PowerShell 프롬프트, macOS/Linux: $SHELL 프롬프트

---

### AC-002: 키보드 입력 전달

**Given** TUI 화면이 활성 셸과 함께 표시된 상태에서
**When** 사용자가 키보드로 `echo hello` 를 입력하고 Enter를 누르면
**Then** 해당 명령이 셸에서 실행되고 `hello` 출력이 패널에 표시되어야 한다

검증 방법:
- 키 입력 -> PTY 전달 -> 셸 실행 -> 출력 표시 전체 플로우 확인
- 한글/CJK 문자 입력 시 깨짐 없이 전달되는지 확인
- 특수 키 (Tab, 방향키, Backspace) 정상 동작 확인

---

### AC-003: 셸 출력 실시간 렌더링

**Given** 활성 셸이 실행 중인 상태에서
**When** 셸에서 대량의 출력이 생성되면 (예: `ls -la /` 또는 `dir`)
**Then** 출력이 실시간으로 패널에 렌더링되어야 하며 UI가 멈추지 않아야 한다

검증 방법:
- 연속 출력 시 UI 프레임 드롭 없음 확인
- 10MB/s 이상의 출력 처리량 벤치마크
- 렌더링 지연 16ms 이하 유지 확인

---

### AC-004: 애플리케이션 정상 종료

**Given** 셸이 실행 중인 상태에서
**When** 사용자가 Ctrl+Q를 누르면
**Then** 모든 셸 프로세스가 정리되고, 터미널이 원래 상태로 복원되어야 한다

검증 방법:
- 종료 후 터미널 상태 정상 확인 (raw mode 해제, alternate screen 복원)
- 고아 프로세스 없음 확인 (ps 또는 tasklist로 검증)
- 종료 시간 1초 이내

---

### AC-005: 셸 비정상 종료 처리

**Given** 셸이 실행 중인 상태에서
**When** 셸 프로세스가 비정상 종료되면 (예: `exit` 명령 또는 강제 종료)
**Then** 애플리케이션이 계속 실행되며, 사용자에게 셸 종료 상태를 표시해야 한다

검증 방법:
- 셸에서 `exit` 입력 후 앱 크래시 없음 확인
- 종료된 셸 패널에 상태 메시지 표시 확인
- 앱 전체 종료 없이 안정적 유지

---

### AC-006: 터미널 크기 대응

**Given** 애플리케이션이 실행 중인 상태에서
**When** 터미널 창 크기가 변경되면
**Then** UI가 새 크기에 맞게 재배치되어야 한다

검증 방법:
- 터미널 리사이즈 시 패널 크기 자동 조정
- 최소 크기(80x24) 미만 시 경고 표시
- 리사이즈 후 셸 출력 깨짐 없음

---

### AC-007: 메모리 사용량 제한

**Given** 단일 셸로 애플리케이션이 실행 중인 상태에서
**When** 일반적인 사용 패턴으로 10분간 사용하면
**Then** 전체 프로세스 메모리가 30MB를 초과하지 않아야 한다

검증 방법:
- 프로세스 메모리 모니터링 (Windows: taskmgr, macOS/Linux: top)
- 대량 출력 후 메모리 증가량 확인
- 메모리 누수 없음 확인 (장시간 실행 테스트)
